### Rachel stricter round-1 review notes

1. **Target:** `PS2 L1–L2` (`gr3-ps2-unit1-lesson1-balanced-and-unbalanced-forces.html`, `gr3-ps2-unit1-lesson2-motion-changes-with-force.html`)  
   **Observation:** The evidence language is much stronger than in many elementary force lessons, but the recorded evidence still leans mostly on broad observations like “moved farther” or “changed motion.” The worksheets could do more to help students compare repeated trials in a more structured way.  
   **Expected revision action:** Tighten worksheet/answer-key language so students are nudged to compare trial-to-trial results more explicitly rather than giving one vague evidence sentence.

2. **Target:** `PS2 L3–L4` (`gr3-ps2-unit1-lesson3-magnets-pull-without-touching.html`, `gr3-ps2-unit1-lesson4-magnetic-patterns.html`)  
   **Observation:** These lessons correctly say that only some materials are magnetic, but the sample object lists still leave room for ambiguous homeschool testing sets. If families happen to choose only clearly magnetic items or only clearly non-metal items, the “some metals, not all metals” idea could land weakly.  
   **Expected revision action:** Add more explicit parent-facing guidance about choosing a mixed test set that includes clearly magnetic objects and clearly non-magnetic metal/nonmetal objects.

3. **Target:** `gr3-ps2-unit1-lesson5-design-a-magnetic-game-or-tool.html`  
   **Observation:** Lesson 5 has a clear design vibe, but the success target is still broad. Students are told to choose a goal, yet the worksheet does not strongly anchor them to a measurable success criterion or a simple design constraint.  
   **Expected revision action:** Revise the worksheet and/or parent notes so the design goal becomes more concrete and testable before students build version 1.

4. **Target:** `gr3-ps2-unit1-lesson6-test-and-improve-your-force-design.html`  
   **Observation:** Lesson 6 appropriately emphasizes evidence and redesign, but the compare-versions section could push harder on what counts as proof that version 2 is better, worse, or the same.  
   **Expected revision action:** Add a more explicit version-1 versus version-2 comparison prompt in the worksheet or answer key so students justify redesign claims with specific test evidence.

5. **Target:** `gr3-ps2-unit1-quiz.html`  
   **Observation:** The quiz captures the major ideas of the unit, but it stays entirely in multiple-choice mode. For a stricter standard, the unit should show at least a little space for evidence-based explanation rather than only recognition.  
   **Expected revision action:** Add one or more short constructed-response opportunities, or otherwise revise the quiz/supporting feedback so students must explain an answer with evidence at least once.

### Margaret stricter round-1 review notes

1. **Target:** `PS2 L1–L2` (`gr3-ps2-unit1-lesson1-balanced-and-unbalanced-forces.html`, `gr3-ps2-unit1-lesson2-motion-changes-with-force.html`)  
   **Observation:** The section labels and shell are aligned well, but the parent-facing answer-key layer could be more skimmable for busy homeschool use. Right now the strongest guidance is present, but some of it is buried in paragraph rhythm rather than quick-scan prompts.  
   **Expected revision action:** Reformat or tighten key answer-key cues so a parent can instantly spot the success criteria, likely evidence statements, and common misconceptions.

2. **Target:** `PS2 L3–L4` (`gr3-ps2-unit1-lesson3-magnets-pull-without-touching.html`, `gr3-ps2-unit1-lesson4-magnetic-patterns.html`)  
   **Observation:** These lessons are conceptually strong, but they use examples like keys and coins that can vary from home to home. Copy-wise, the lesson should signal that mixed results are expected and scientifically useful, not confusing.  
   **Expected revision action:** Clarify the parent notes and sample wording so variable household objects do not feel like a lesson failure.

3. **Target:** `gr3-ps2-unit1-lesson5-design-a-magnetic-game-or-tool.html`  
   **Observation:** The design lesson still feels a bit more open-ended than the surrounding investigation lessons. That is fun, but it needs a little more LLC parent-support language so families know what “good enough” planning looks like before build time.  
   **Expected revision action:** Add more explicit planning guidance or a mini exemplar in the worksheet/answer-key layer for the intended level of detail.

4. **Target:** `gr3-ps2-unit1-lesson6-test-and-improve-your-force-design.html`  
   **Observation:** The redesign lesson is structurally solid, but the worksheet asks for improvement reflection in fairly broad blanks. A stronger compare-and-improve frame would make the engineering arc easier for a parent to coach.  
   **Expected revision action:** Strengthen the worksheet prompts so version 1, change made, and result in version 2 are clearly separated and easy to compare.

5. **Target:** `gr3-ps2-unit1-quiz.html`  
   **Observation:** The quiz shell works, but it visually diverges from the lesson pages more than the rest of the unit does. It also lacks a parent-facing answer-key/reference layer that matches the family-teaching pattern used elsewhere in the unit.  
   **Expected revision action:** Improve LLC shell parity for the quiz and add clearer parent-facing review support so the quiz feels like part of the same product, not a separate widget.
