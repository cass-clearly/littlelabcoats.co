Test and Improve Your Bridge

Engineering — Bridge Builders | Unit 1, Lesson 5 of 6

📚 Grade: 2nd Grade⏱️ Duration: 40–50 minutes🛠️ Subject: Engineering — Bridge Builders
🪙📈🔧
Testing shows where a bridge is strong, where it struggles, and what to improve next.
📋 NGSS Standard:
K-2-ETS1-3 — Analyze data from tests of two objects designed to solve the same problem to compare the strengths and weaknesses of how each performs.

🎯 Learning Objective

Students will run a fair bridge test, identify a weak point, and improve version 1 based on evidence from the test.

💡 The Big Idea

Testing is how engineers learn. A bridge is not “good” just because it looks nice. It needs evidence from a fair test.

🔬 The Science

A fair test keeps the important conditions the same while engineers look at how the design performs. For a bridge, that means the same gap and the same type of weight each time.

Students test version 1 by adding weight carefully and watching what happens first: sagging, slipping, bending, or staying strong.

After the test, engineers make one purposeful change instead of changing everything at once. That way they can tell whether the redesign helped.

Let's Get Started

Step 1 — Set Up a Fair Test: Use the same gap and the same type of weights for every trial.
Step 2 — Test Version 1: Add weight slowly and record when the bridge starts to sag, bend, or fail.
Step 3 — Find the Weak Point: Name the part that struggled first and describe what happened.
Step 4 — Make One Smart Change: Adjust one feature such as folded sides, extra support, or stronger tape placement.
Step 5 — Test Version 2: Run the same test again and compare version 2 to version 1 with evidence.

Discussion Questions

What made this a fair test?Look for: same gap, same kind of weights, same challenge each time.
What was the weak point in version 1?Look for: a real observed weakness, not a guess.
Why should we make one change at a time?Look for: so we can tell what change helped.
How do you know version 2 is better?Look for: the answer must use evidence from the test.

Extensions

Extension: Try one more redesign only if you can keep the test fair and compare it honestly.

Extension: Graph version 1 and version 2 by drawing bars for how much weight each one held.

Parent/Teacher Notes

Teaching note: If a bridge fails quickly, frame that as useful information, not as a mistake.

Teaching note: Push students to name the exact weak point rather than saying the whole bridge was “bad.”

✂️ PARENT ANSWER KEY — Detach before giving worksheet to students

1️⃣ Fair Test Record — What to Look For

Students should record version 1 and version 2 under the same testing conditions and note what happened in each trial.

2️⃣ Weak Point and Smart Change — Strong Thinking

Look for a clear weak point, one specific redesign, and a conclusion tied to the test evidence such as more weight held or less sagging.

✅ What Success Looks Like

  • Student compares version 1 and version 2 fairly.
  • Student identifies a real weak point from version 1.
  • Student explains whether the redesign helped using evidence.
