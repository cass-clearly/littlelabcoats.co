Build and Run Your First Filter Trial

Engineering — Clean Water Design Lab | Unit 1, Lesson 4 of 6

📚 Grade: 5th Grade⏱️ Duration: 50–60 minutes🛠️ Subject: Engineering — Clean Water Design Lab
🛠️💧📊
Version 1 is for learning, not perfection.
📋 NGSS Standard:
3-5-ETS1-1 / 3-5-ETS1-2 / 3-5-ETS1-3 — Define a design problem that reflects a need or a want, includes specified criteria for success and constraints on materials, time, or cost; generate and compare multiple possible solutions; and plan and carry out fair tests with variables controlled.

🎯 Learning Objective

Students will build version 1 of a filter prototype, run a fair trial, and collect evidence about clarity, flow, and structural performance.

💡 The Big Idea

The first build is a learning tool. Engineers collect evidence from version 1 so redesign decisions are based on results, not guesses.

🔬 Engineering Thinking

Prototype testing should generate useful information. Students should focus on collecting observations that connect directly to the criteria from Lesson 1.

In Grade 5, it is reasonable to track more than one performance feature at once. Students can record a clarity score, note how long the flow took, and describe whether the structure stayed stable.

Students should understand that failure is informative. If the filter leaks, clogs, tips, or lets many particles through, those observations reveal where improvement is needed.

Let's Get Started

Step 1 — Build version 1: Students assemble the planned prototype as closely as possible to the drawing.
Step 2 — Prepare the test: Use the same amount of dirty water and the same setup for each group or trial.
Step 3 — Run trial 1: Pour carefully, observe flow, and collect the filtered water.
Step 4 — Record evidence: Note clarity, time, leaks, clogging, tipping, or any visible weak point.
Step 5 — Make an initial claim: Students explain one strength and one weakness of version 1 using evidence.

Discussion Questions

Why is version 1 useful even if it does not work well?It reveals what needs to improve.
What counts as evidence in this test?Observed clarity, timing, leaks, clogging, and stability.
Why should every group use the same amount of dirty water?That keeps the test fair.
Which criterion seems hardest to meet so far?Students should connect to actual test results.

Extensions

Extension: Create a simple 1–5 clarity scale and compare scores across groups.

Extension: Ask students whether the prototype is better suited for clarity or speed and defend the claim.

Parent/Teacher Notes

Teaching note: Keep safety and sanitation language clear. This is model dirty water only.

Upward-rigor move: Encourage students to use evidence phrases such as “Our trial showed...” or “The data suggest...” in their oral explanations.

✂️ PARENT ANSWER KEY — Detach before giving worksheet to students

1️⃣ Record Version 1 Results — Sample Use

Accept varied outcomes as long as students connect them to observations such as clearer water, slow flow, clogging, or tipping.

2️⃣ Strength and Weakness — Sample Reasoning

A strong response names one criterion the design met better and one criterion that still needs work.

3️⃣ Sketch or Note the Weak Point — Sample Use

Look for specific details such as where the filter leaked, sagged, overflowed, or let many particles through.

✅ What Success Looks Like

  • Student records evidence tied to criteria.
  • Student identifies a specific weak point in version 1.
  • Student uses data or observations instead of opinion alone.

🛠️ Troubleshooting / Teaching Notes

  • If students only write “it worked” or “it failed,” ask what exactly happened and where.
  • If students change the setup mid-test, remind them that version 1 evidence must come from the planned fair trial.
