🔁 Redesign Using Test Evidence

Grade 5 Engineering — Clean Water Design Lab Quick Reference
Big Idea

Version 2 should be built from evidence, not guessing.

Key Vocabulary
  • iteration: a new version made after testing
  • justify: explain with evidence and reasoning
  • weak point: the part of the design that needs work
  • trade-off: a possible downside created by a change
Remember This
  • Change one or two key features on purpose.
  • Use version 1 evidence to explain each redesign decision.
  • Version 2 still needs a fair comparison test.
Use It to Explain
  • Start with: We changed ___ because...
  • Add: The evidence from version 1 showed...
  • Finish with: We predict version 2 will...
